# 卡片生成模块 - 详细规格说明

## 1. 模块概述

### 1.1 模块名称
卡片生成模块（Card Generation Module）

### 1.2 模块目标
生成包含AI诗句和二维码的分享卡片，支持导出和分享功能。

### 1.3 模块定位
作为应用的分享核心模块，为用户提供美观的分享卡片。

## 2. 功能需求

### 2.1 核心功能
- **卡片生成**：使用Canvas绘制包含AI诗句和二维码的卡片
- **二维码生成**：生成包含share_id的二维码
- **表情包选择**：从三类表情包文案中选择一个叠加到卡片
- **卡片导出**：导出为PNG图片
- **卡片分享**：使用系统分享API分享卡片
- **分享链接生成**：生成包含share_id的分享链接

### 2.2 业务规则
- **卡片布局**：背景/配色/字体/留白统一
- **二维码规范**：携带share_id，用于微信/小红书扫码
- **表情包选择**：默认选"安慰"，用户可在卡片预览中选择
- **导出限制**：PNG格式，体积≤300KB，不携带EXIF
- **分享兜底**：不支持系统分享时提供"保存图片/复制链接/复制文案"兜底
- **离线处理**：网络不可用时显示"离线分享"占位

## 3. 技术实现

### 3.1 技术栈
- **前端框架**：Vue.js / React
- **Canvas API**：用于绘制卡片
- **QRCode库**：用于生成二维码
- **Web Share API**：用于系统分享
- **File API**：用于导出图片

## 6. 风险与应对

### 6.1 风险
- **Canvas绘制失败**：浏览器不支持Canvas或绘制出错
- **导出失败**：图片导出失败或体积过大
- **分享失败**：系统分享API不可用
- **二维码生成失败**：QRCode库加载失败

### 6.2 应对措施
- **降级方案**：提供图片占位或文本分享兜底
- **错误处理**：捕获并处理所有可能的错误
- **性能优化**：优化Canvas绘制和图片处理
