# 📄 话说APP 产品需求文档（PRD）｜V1.0（研发交付版）

> **适用对象**：新人独立开发 / 外包研发对接  
> **核心差异化**：AI情绪加密（脱敏翻译）× 微信/小红书跨域匿名回声闭环  
> **形态**：手机优先 Web（PWA）+ 极简后端  
> **技术栈**：前端PWA（移动Web）/ Supabase（分享计数存储）/ OpenRouter（AI翻译）/ 极简后端API（保护密钥与限流）

---

## 1. 项目概述

### 1.1 项目名称
话说APP（Huashuo）

### 1.2 产品定位
社恐/高敏感人群专属·零压力匿名情绪记录与加密分享工具。

### 1.3 核心用户痛点
- 不敢在微信/微博/小红书直说脆弱：怕被熟人认出、怕被评价、怕被过度关心
- 既渴望被接住，又排斥强社交与即时沟通压力

### 1.4 核心卖点（一句话）
把“真实心事”留在本地，把“情绪翻译成高级社交货币”，让熟人用“扫码送拥抱”的方式给你一个匿名回声。

### 1.5 V1.0 目标与边界
**V1.0 目标**
- 跑通 Encrypted Echo 闭环：PWA记录 → AI加密卡片 → 分享到微信/小红书 → 网页扫码送拥抱 → 回流到PWA
- 以“快速验证闭环”为优先：允许放弃部分本地隐私承诺（不再承诺“清理不可恢复/永不丢失”）

**V1.0 明确不做（避免失控）**
- 不做应用内公开内容流/同频共鸣池（不做社区）
- 不做评论、私信、关注、点赞排行、在线状态
- 不做复杂账号体系（送拥抱无需登录）
- 不做审核后台（V1.0无公开UGC社区流、且网页端无输入框）

### 1.6 术语与约定
- **mood_band（情绪频段）**：仅 3 档，用于分类与视觉基调
  - 🌧️ = 0（低落/焦虑/疲惫）
  - 🌤️ = 1（平静/波动/复杂）
  - ☀️ = 2（轻松/积极/被接住）
- **share_id**：分享卡片唯一ID（UUID v4），二维码与网页路由均以此为主键
- **拥抱（hug）**：网页端唯一交互，表现为计数增长；V1.0 不做评论/文本输入
- **本机存储**：默认存放在浏览器本地（IndexedDB/Cache），用户清理浏览器数据可能导致历史丢失

## 2. 核心闭环（Encrypted Echo）

### 2.1 用户动线
```
PWA：选择/切换匿名身份 → 写下心事（默认本机存储） → AI自动选择情绪频段 → AI情绪加密（翻译成诗句） → 卡片自动布局在正文后 → 分享到微信/小红书
网页：熟人扫码 → 只看到诗句 → 点击“送出拥抱”
PWA：同步拥抱数 → 展示“来自远方的匿名拥抱”
```

### 2.2 关键设计原则
- 原始心事默认仅存本机：不作为“必须上云”的依赖；但不做强隐私承诺（可能因清缓存而丢失）
- 分享卡片只包含：AI诗句、情绪风格、二维码（share_id）
- 网页端无输入框：只允许点击“拥抱”，从源头降低合规/攻击面

## 3. 信息架构与页面清单

### 3.1 PWA 页面（路由建议）
- 身份切换/管理（列表、创建、归档/清理）
- 首页/列表（按时间、按频段筛选、已归档筛选）
- 发布/编辑（纯文字；频段由AI自动选择）
- 详情页（原文仅自己可见；卡片自动紧随正文展示；导出/分享、拥抱数）
- 回声页（按卡片聚合：哪个分享收到多少拥抱）
- 设置（隐私说明、本机存储提示、清理数据、一键导出备份、反馈入口）

### 3.2 分享落地页（同域网页）
- `/s/<share_id>`：展示 AI诗句 + “送出拥抱”按钮 + 动效反馈

## 4. 功能需求（模块化）

### 4.1 本机身份系统（P0）
**功能点**
- 创建身份：2-8字昵称 + Emoji；首次进入自动创建默认身份（存浏览器本地）
- 身份切换：切换后内容严格隔离
- 数据硬隔离：跨身份查询必须返回空
- 归档/清理：支持归档与清理；清理为删除本机数据，不承诺不可恢复

**验收**
- 离线可打开与浏览已缓存内容；切换身份后列表/详情不串号

### 4.2 情绪树洞（本地记录）（P0）
**功能点**
- 发布：纯文字；频段由AI自动选择（🌧️/🌤️/☀️）
- 频段纠错：用户可在发布后对频段手动修改 1 次（用于纠正AI误判），修改后锁定不可再改
  - 入口：详情页频段标签旁“纠错”按钮；已纠错后按钮置灰
  - 行为：纠错会触发“重新生成卡片”（重新调用AI，刷新 ai_poem 与 stickers[3]；并更新 mood_band_final）
- 浏览：按时间排序；按频段筛选；已归档筛选
- 管理：归档/删除/清理本机数据
- 脱敏提示：检测疑似手机号/地址并提醒（不强拦截）

**异常**
- 本机存储写入失败：提示“未保存”，不进入卡片生成流程

### 4.3 AI情绪加密翻译（P0）
**目标**
将原始日记翻译为“可公开分享的社交货币文案”，避免暴露隐私与具体细节。

**输入/输出约束**
- 输入：仅用户文本 + 风格（默认：治愈现代诗）
- 输出：
  - `mood_band(🌧️/🌤️/☀️)`：AI建议频段（用户允许手动纠错一次）
  - `ai_poem`：≤15字；不包含人名、地点、手机号、具体事件细节；不含标点可选
  - `stickers[3]`：情绪跨度明显的 3 个“表情包文案”（每条≤10字，附 1 个Emoji），固定三类：
    - 安慰
    - 吃瓜
    - 损友式诋毁（必须是“吐槽式打气”，不能攻击人格、不能羞辱、不能诱导自伤）

**内容安全约束（V1.0）**
- 诗句与表情包文案不得包含：辱骂脏话、人身攻击、地域/性别/种族歧视、诱导自伤、自杀相关细节
- “损友式诋毁”仅允许“吐槽式打气”，示例方向：夸张吐槽 + 站你这边（但不贬低你的人格）

**降级**
- AI超时/失败：使用本地短句模板库替代（保证可用）
  - 本地模板同样需要返回：mood_band + ai_poem + stickers[3]

### 4.4 卡片生成与导出分享（P0）
**功能点**
- Canvas 生成卡片：背景/配色/字体/留白统一
- 二维码：携带 `share_id`，用于微信/小红书扫码
- 自动布局：详情页中卡片自动紧随“心事正文”后展示（生成中显示占位态）
- 表情包三选一：默认选“安慰”，用户可在卡片预览中从三条表情包文案中选择 1 条叠加到卡片底部（可关闭不显示）
- 导出与分享：
  - 导出：PNG（网页生成图片不携带EXIF）；体积≤300KB
  - 分享：优先使用系统分享（Web Share API）；不支持时提供“保存图片/复制链接/复制文案”兜底

**二维码与分享链接规范**
- 二维码内容为 URL：`https://<你的域名>/s/<share_id>`
- share_id 必须不可预测（UUID v4），避免被枚举
- 若网络不可用或写入分享记录失败：仍允许导出卡片，但卡片底部二维码区域显示“离线分享”占位（不生成二维码）

**验收**
- 移动端浏览器导出/分享可用；在微信内置浏览器限制下仍有兜底（至少能复制链接/长按保存）
- 频段纠错：同一条心事最多修改 1 次；超过次数入口置灰且不影响已有卡片展示/导出
- 表情包：三类文案必定存在；切换选择后再次进入详情页仍保持上次选择

### 4.5 跨域匿名回声（网页送拥抱）（P0）
**网页端**
- 展示：只展示 AI诗句
- 操作：点击“送出拥抱”（无登录、无输入）
- 动效：点击后轻动效反馈

**防刷（最小实现）**
- LocalStorage 限次：同设备对同一 share_id 每日最多 1 次
- 服务端限流：同IP对同一 share_id 每分钟最多 3 次（超出返回 429）
- 数据最小化：分享页不采集账号信息；不记录用户可识别身份，仅用于防刷的匿名标识（如本地生成的随机device_id）

**PWA端回流**
- 冷启动或进入回声页时，批量拉取 share_id 的 `hugs_count`
- 若计数增加：提示“收到X个拥抱”

## 5. 数据与存储设计

### 5.1 浏览器本机存储（建议表/对象）
- identities：`id`、`nickname`、`emoji`、`created_at`
- posts：`id`、`identity_id`、`mood_band_ai`、`mood_band_final`、`mood_band_edit_count(0/1)`、`content_text`、`created_at`、`archived_at?`、`deleted_at?`
- cards：`id`、`post_id`、`share_id?`、`ai_poem`、`sticker_comfort`、`sticker_gossip`、`sticker_roast`、`sticker_selected_type?`、`hugs_count`、`created_at`、`exported_at?`、`synced_at?`

### 5.2 Supabase（仅分享数据上云）
表：`shared_cards`
- 字段：`share_id(uuid, pk)`、`ai_poem(text)`、`mood_band(int)`、`hugs_count(int, default 0)`、`created_at(timestamp)`、`updated_at(timestamp)`、`last_hug_at(timestamp?)`
- 约束：`mood_band in (0, 1, 2)`；`hugs_count >= 0`

RLS口径（最小化）
- 读取：允许按 `share_id` 读取单条（分享页展示需要）
- 写入：仅后端服务可插入新卡片（只写 share_id/ai_poem/mood_band）
- 更新：不开放任意 update；只允许通过 RPC 做受控自增（同时做限流与防刷）

## 6. 接口与事件（最小集合）

### 6.1 AI翻译（PWA → 后端 → OpenRouter）
后端转发调用 OpenRouter，避免在前端暴露密钥，并在后端统一做限流与内容安全兜底。
- 请求：text + 风格
- 响应：mood_band + ai_poem + stickers[3]

**建议响应结构（示例）**
```json
{
  "mood_band": 1,
  "ai_poem": "把心放回呼吸里",
  "stickers": {
    "comfort": "抱抱你 🫂",
    "gossip": "我先吃个瓜 🍉",
    "roast": "你又在内耗啦 🤺"
  }
}
```

### 6.2 分享上云（PWA → 后端 → Supabase）
- create：创建 share_id，并写入 `shared_cards`（仅 ai_poem/mood_band）

### 6.3 送拥抱（网页 → 后端 → Supabase）
- update/RPC：`hugs_count = hugs_count + 1`（带限流与防刷）

### 6.4 拥抱回流（PWA → 后端 → Supabase）
- select：按 share_id 批量读取 hugs_count

### 6.5 极简后端 API 契约（推荐）
**说明**
- 所有接口只处理“加密后的诗句与计数”，不接收/不存储/不记录用户原始心事内容。
- 后端必须避免将 `text` 写入任何日志（包括错误日志），只记录请求ID与耗时等无内容信息。

#### 6.5.1 POST `/api/ai/translate`
用途：将心事文本翻译为 `mood_band + ai_poem + stickers`（用于卡片与频段）。

请求体：
```json
{
  "text": "string",
  "style": "heal_poem"
}
```

响应体（200）：
```json
{
  "mood_band": 1,
  "ai_poem": "把心放回呼吸里",
  "stickers": {
    "comfort": "抱抱你 🫂",
    "gossip": "我先吃个瓜 🍉",
    "roast": "你又在内耗啦 🤺"
  }
}
```

状态码：
- 200：成功
- 400：参数不合法（text 为空/过长）
- 429：限流（同IP/同设备请求过频）
- 502：上游AI失败（前端需降级为本地模板）

#### 6.5.2 POST `/api/share/create`
用途：创建 share_id，并把“分享页所需最小数据”写入 Supabase（不含原文）。

请求体：
```json
{
  "ai_poem": "string",
  "mood_band": 1
}
```

响应体（200）：
```json
{
  "share_id": "uuid",
  "share_url": "https://<你的域名>/s/<share_id>"
}
```

状态码：
- 200：成功
- 400：参数不合法（mood_band 非 0/1/2 等）
- 429：限流
- 502：写入失败（前端进入“离线分享”降级：无二维码）

#### 6.5.3 POST `/api/share/hug`
用途：给指定 share_id 增加 1 个拥抱（后端做防刷与限流）。

请求体：
```json
{
  "share_id": "uuid",
  "device_id": "string"
}
```

响应体（200）：
```json
{
  "hugs_count": 12
}
```

状态码：
- 200：成功
- 404：share_id 不存在或已失效
- 429：限流/防刷触发（同设备/同IP重复点击）
- 502：写入失败

#### 6.5.4 POST `/api/share/batch`
用途：PWA 批量拉取多个 share_id 的 `hugs_count`（用于回流页/列表页刷新）。

请求体：
```json
{
  "share_ids": ["uuid1", "uuid2"]
}
```

响应体（200）：
```json
{
  "items": [
    { "share_id": "uuid1", "hugs_count": 12 },
    { "share_id": "uuid2", "hugs_count": 3 }
  ]
}
```

状态码：
- 200：成功
- 400：参数不合法（数组为空/过长）
- 429：限流

## 7. 非功能性需求
- 性能：列表60fps；卡片生成≤2s；导出≤2s（视机型）
- 隐私：心事默认本机存储；分享仅诗句；不采集账号信息
- 安全：网页端无输入框；后端限流防刷；后端不记录原文日志
- 可用性：AI/网络失败有降级（本地模板），仍可生成并分享（无二维码占位）

### 7.1 兼容性与发布（V1.0）
- 浏览器：主流移动端浏览器（Chrome/Safari）与微信内置浏览器（能力受限但保证可用兜底）
- 安装：支持“添加到主屏幕”（PWA）；不依赖应用商店
- 部署：前端静态托管 + 极简后端API（建议同域，避免跨域与微信内置浏览器问题）

## 8. 指标与验收

### 8.1 指标
- 卡片分享率 ≥15%
- 单卡平均拥抱数 ≥3
- 回流触达率（分享后用户再次打开PWA看到拥抱）≥30%

### 8.2 验收清单（研发自测）
- 身份隔离：切换身份内容严格隔离
- AI频段：发布后能看到 AI 建议频段；允许纠错 1 次；纠错后入口置灰
- AI降级：断网/超时仍能出卡片（本地模板返回 mood_band + ai_poem + stickers）
- 卡片布局：详情页“正文下方”自动展示卡片；生成中有占位态；生成完成自动替换
- 表情包：三类文案都存在；默认选安慰；可切换/可关闭；选择会持久化
- 分享链路：导出卡片（含二维码）→ 微信扫码 → 网页送拥抱 → PWA回流计数正确
- 离线分享：网络断开或上云失败时仍能导出卡片（无二维码占位），不崩溃
- 防刷：同设备对同卡每日只计 1 次；服务端限流触发时分享页能看到友好提示（429）

## 9. 版本规划
- V1.0：闭环跑通（AI加密卡片 + 网页送拥抱 + 回流）
- V1.1：AI人格/风格选择、回声动效强化、离线重试队列
- V1.2：如引入应用内社区流，需新增举报/审核与合规体系
